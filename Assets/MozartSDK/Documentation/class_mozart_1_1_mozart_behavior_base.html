Try every possible way imaginable to find the MozartManager in the scene. Some development teams do not like Singletons due to tight coupling problems, so this is a replacement for just calling MozartManager.instance all over the codebase. If you want to replace MozartManager with a custom manager you can change how GetManager works here and implement an interface instead.
